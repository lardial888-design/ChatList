# PLAN

Ниже поэтапный план реализации программы ChatList на основе `PROJECT.md`.

## Этап 1. Подготовка проекта
- Создать структуру файлов: `main.py`, `db.py`, `models.py`, `network.py`.
- Настроить зависимости (`PyQt`, `requests`/`httpx`, `python-dotenv` при необходимости).
- Подготовить файл `.env` с API-ключами.

## Этап 2. Слой данных (db.py)
- Создать подключение к SQLite.
- Реализовать инициализацию БД и создание таблиц.
- Добавить базовые CRUD-операции для `prompts`, `models`, `results`, `settings`.

## Этап 3. Слой моделей (models.py)
- Реализовать получение списка активных моделей.
- Описать структуру модели для запроса: имя, API URL, имя переменной ключа.

## Этап 4. Сетевой слой (network.py)
- Реализовать отправку промта в конкретную модель.
- Поддержать разные провайдеры (OpenAI, DeepSeek, Groq и др.) через единый интерфейс.
- Обработать ошибки сети и таймауты.

## Этап 5. Интерфейс (main.py)
- Реализовать форму ввода промта и список сохраненных промтов.
- Реализовать таблицу результатов с колонками: модель, ответ, selected (чекбокс).
- Добавить кнопки: "Отправить", "Сохранить", "Новый запрос".

## Этап 6. Логика рабочего процесса
- При отправке промта: получить активные модели, отправить запросы, заполнить временную таблицу.
- При сохранении: записать выбранные строки в `results`, очистить временную таблицу.
- При новом запросе: очистить временную таблицу и начать новый цикл.

## Этап 7. Дополнительные функции (опционально)
- Поиск и сортировка во всех таблицах.
- Экспорт выбранных результатов в Markdown/JSON.
- Логирование запросов.

## Этап 8. Проверка и тестирование
- Проверить корректность сохранения данных.
- Проверить обработку ошибок API.
- Проверить работу интерфейса при большом количестве данных.
